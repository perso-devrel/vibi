## 구조: Task + Agent Loop 2단 루프

### 1단: Task (Phase 관리)

시작 시 아래 Phase들을 Task로 등록하고 진행 상황을 추적해.
이미 테스트가 존재하고 통과하는 Phase는 완료로 표시해.
BFF(서버) 담당 기능은 제외하고 Android 앱 로직만 구현한다.

| Task | Phase | 내용                                             |
|------|-------|------------------------------------------------|
| 1 | 입력 | 갤러리 영상 선택, 포맷/용량 검증                            |
| 2 | 자막 편집 | SubtitleSegment 모델, 세그먼트 CRUD, 위치 편집, undo/redo |
| 3 | 내보내기 | SRT/ASS 직렬화, ffmpeg-kit 번인, 프로그레스              |
| 4 | 저장/공유 | 갤러리 저장, SRT 내보내기, 공유 시트                        |

### 2단: Agent Loop (TDD 사이클)

현재 진행 중인 Task 내에서 아래 사이클을 반복해:

1. 기존 테스트 실행 → 현재 상태 파악
2. 누락된 기능에 대해 실패하는 테스트 작성
3. 최소 코드로 테스트 통과시키기
4. 리팩토링
5. 모든 테스트 통과 확인 → Task 완료 처리 → 다음 Task로 이동

### 규칙

- 한 번에 하나의 Task만 진행 (현재 Task의 테스트가 모두 통과해야 다음으로)
- 테스트 실행은 변경 관련 테스트만 선택적으로 (`--tests` 필터 사용)
- Phase 전환 시 전체 유닛 테스트 한 번 실행하여 회귀 확인
- 모든 Task 완료 시 <promise>COMPLETE</promise> 출력
- 해결할 수 없는 에러 발생 시에도 <promise>COMPLETE</promise> 출력

### 제외 범위 (BFF 서버 담당)

아래는 BFF 서버에서 구현할 사항으로 이 루프에서 다루지 않는다:
- 파이프라인: Perso API 오케스트레이션 (업로드→번역→폴링→다운로드→스크립트)
- 재생성 요청: Request Translation 재호출, 크레딧 차감/관리
- 세션/결제/크레딧 관리
- 서버사이드 ffmpeg 폴백 렌더
